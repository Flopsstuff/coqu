## Why

Сейчас в API нет структурированного логирования — только `console.log` в паре мест. Нет возможности отслеживать что происходит на бэкенде: создание проектов, клонирование, ошибки, авторизацию. Нужна система логов с удобным просмотром в UI, чтобы администратор видел активность и мог диагностировать проблемы.

## What Changes

- Добавить серверную библиотеку логирования с уровнями (info, warn, error) и структурированными полями (category, projectId, agentId и т.д.)
- Запись логов в JSON-файлы с ротацией по дням (один файл на день, автоочистка старых)
- REST API для чтения логов с фильтрацией по уровню, категории, дате и пагинацией
- Новая страница `/logs` в веб-интерфейсе для просмотра и фильтрации логов
- Инструментировать существующие операции API: auth, projects, agents, git clone

## Capabilities

### New Capabilities
- `backend-logging`: Серверная система логирования — запись в файлы, ротация, уровни, категории
- `logs-api`: REST-эндпоинты для чтения и фильтрации логов
- `logs-ui`: Страница просмотра логов в веб-интерфейсе

## Impact

- **`packages/api`**: Новый модуль логирования, новые эндпоинты `/api/logs`, инструментирование существующих хэндлеров
- **`packages/shared`**: Новые типы `LogEntry`, `LogLevel`, `LogCategory`, `LogsQuery`, `LogsResponse`
- **`packages/web`**: Новая страница `LogsPage`, маршрут `/logs`, пункт в навигации
- **Файловая система**: Директория для лог-файлов (настраиваемая, по умолчанию `./logs/`), JSON-формат, ротация по дням
- **Зависимости**: `pino` + `pino-roll` для записи логов (transport в worker thread), свой код для чтения через API
